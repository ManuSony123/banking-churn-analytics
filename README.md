# Banking Customer Churn Analytics Project

## Project Overview
End-to-end data analytics project covering ETL, data warehousing,
SQL analytics, and Power BI dashboards for a retail banking churn use case.

## Folder Structure
```
banking_churn_analytics/
├── config.py                          ← Central config — update DB_PASSWORD here
├── run_all_phases.py                  ← Master script to run everything
├── data/
│   ├── raw/                           ← Place BankChurners.csv here
│   ├── staging/                       ← Auto-generated by Phase 3
│   └── warehouse/                     ← Auto-generated by Phase 5
├── scripts/
│   ├── phase1_step1_test_connection.py
│   ├── phase1_step2_validate_raw_data.py
│   ├── phase2_step1_eda_univariate.py
│   ├── phase2_step2_eda_bivariate.py
│   ├── phase3_step1_generate_supporting_tables.py
│   ├── phase3_step2_etl_transformations.py
│   ├── phase3_step3_scd2_incremental.py
│   ├── phase4_load_warehouse.py
│   └── phase5_analytics_queries.py
├── sql/                               ← Place SQL scripts here
├── powerbi/                           ← Place .pbix file here
└── docs/                              ← Reports and EDA plots saved here
```

## Setup Instructions

### 1. Install dependencies
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate      # Mac/Linux
pip install pandas numpy matplotlib seaborn jupyter sqlalchemy pymysql openpyxl
```

### 2. Download dataset
- Go to: https://www.kaggle.com/datasets/radheshyamkollipara/bank-customer-churn
- Download BankChurners.csv
- Place in: data/raw/BankChurners.csv

### 3. Configure database
- Open config.py
- Update DB_PASSWORD with your MySQL root password

### 4. Create MySQL database
Run in MySQL Workbench:
```sql
CREATE DATABASE banking_churn_db CHARACTER SET utf8mb4;
```

### 5. Run the project
```bash
# Run all phases in sequence
python run_all_phases.py

# OR run individual phases
python scripts/phase1_step1_test_connection.py
python scripts/phase1_step2_validate_raw_data.py
python scripts/phase2_step1_eda_univariate.py
python scripts/phase2_step2_eda_bivariate.py
python scripts/phase3_step1_generate_supporting_tables.py
python scripts/phase3_step2_etl_transformations.py
python scripts/phase3_step3_scd2_incremental.py
python scripts/phase4_load_warehouse.py
python scripts/phase5_analytics_queries.py
```

## Technologies Used
- Python (Pandas, NumPy, Matplotlib, Seaborn, SQLAlchemy)
- MySQL (Star Schema, Indexes, Window Functions)
- Power BI (3-page dashboard)
- ETL concepts: SCD Type 2, Deduplication, Null Treatment, Derived Columns

## Star Schema
- dim_customer (SCD Type 2)
- dim_product
- dim_date
- dim_churn_reason
- fact_transactions (central fact table)
